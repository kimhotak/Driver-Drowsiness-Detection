📋 프로젝트 개요

진행 인원: 5인 팀 프로젝트

내용: 현대오토에버 스마트팩토리 2기 과정 중 수행한 프로젝트로, 운전자의 상태를 실시간으로 모니터링하여 눈 감김 여부로, 졸음운전 위험을 감지하고 경고하는 시스템입니다.

## 🛠 작업 내용

### 1. 전략 패턴 기반의 인터페이스 설계

팀원들이 만든 다양한 알고리즘(CNN, MLP, RF 등)을 유연하게 교체하며 테스트하기 위해 전략 패턴을 적용했습니다. [EyeCloseDetector](./eyeCloseDetection/EyeCloseDetection.py)라는 추상 클래스를 설계하여, 상단 로직을 수정하지 않고도 모델을 자유롭게 바꿀 수 있는 구조를 만들었습니다.

### 2. 이미지 전처리 공통화

전체 프레임에서 눈 영역을 찾아 90x90 크기로 정규화(Crop & Resize)하는 과정을 모듈화했습니다. 각 모델이 복잡한 이미지 처리 대신 '예측'에만 집중할 수 있게 정리했습니다.

### 3. 프로젝트 일정 관리 및 조율

팀 내 전체적인 개발 일정 관리와 태스크 조율을 담당했습니다. 작업 우선순위를 설정하여 개발 병목 현상을 방지하고자 했습니다.

### 4. 눈 감김 감지 모델 구현

적외선 이미지 기반의 눈 감김 감지 모델(Infrared MLP) 중 하나를 직접 구현하고 학습을 진행했습니다

## 🔍 시행착오와 배운 점

### 테스트 과정에서 발견된 데이터 규격 문제

설계 단계에서 전처리 규격(90x90, BGR)을 미리 약속했지만, 실제 모델을 합쳐보니 RuntimeError가 자주 발생했습니다. 전처리 로직은 약속대로 구현되었으나, 정작 모델을 가져다 쓰는 과정에서 약속과 다른 이미지나 형식을 사용하면서 문제가 생긴 것이었습니다. 이런 오류들이 컴파일이나 빌드 시점이 아닌, 실제 성능 테스트(런타임) 도중에야 발견되면서 일정이 조금씩 지연되었습니다. 파이썬의 높은 자율성이 대규모 협업이나 안정성이 중요한 시스템에서는 오히려 관리의 어려움이 될 수 있다는 것을 직접 경험했습니다.

### 기술적 성찰

단순한 추상화뿐만 아니라, abc 모듈의 엄격한 활용과 Type Hinting을 통해 인터페이스를 기술적으로 강제하는 것이 얼마나 중요한지 깨달았습니다.

대규모 협업에서는 문서상의 약속에만 의존하기보다, 런타임 에러를 사전에 방지할 수 있는 정적 분석 도구(mypy 등)나 강제 장치를 설계 단계부터 고민해야 함을 배웠습니다.
